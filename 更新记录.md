# 更新记录

## 1.0.3 - 2026-03-23

### 重点更新

- 工具栏新增 `解包格式化`、`解包压缩` 两个按钮，专门处理“根节点是字符串、字符串内部仍然是一份 JSON”的数据。
- 工具栏里的 `压缩 / 下载 / 复制 / 导入` 收成两列紧凑小按钮组，减少右侧纵向占用。

### 字符串 JSON 解包

- 新增显式解包逻辑：只有用户点击解包按钮时，才会把包装字符串改写成正常的对象/数组 JSON。
- 常规 `格式化 / 压缩` 继续保持“输入是什么就处理什么”的语义，不会自动篡改普通字符串数据。
- 当当前内容不属于可解包的 JSON 字符串时，会给出轻提示而不是静默失败。
- 补充了独立的最小测试，覆盖解包识别、格式化输出、压缩输出和非法场景。

### 工具栏整理

- `压缩 / 下载 / 复制 / 导入` 改为 2x2 紧凑按钮布局，视觉风格更接近现有的分段小按钮。
- 保持原有按钮 id 和交互逻辑不变，只调整编排和尺寸，避免影响既有事件绑定。

## 1.0.2 - 2026-03-20

### 重点更新

- 树形预览的横向滚动与右侧路径固定逻辑重新收敛，避免修复过程中把滚动条越撑越长。
- 底部“当前节点”摘要重新接上完整长字符串补全，不再直接显示树列表里被截断过的预览文本。

### 树形预览与横向滚动

- 预览虚拟列表新增“当前片段最大横向内容宽度”基线，同一批可见行会共享这条宽度，短值行与长值行在横向滚动时不再出现右侧路径表现不一致的问题。
- 树形路径样式收敛为更稳定的 `sticky + 实色背景 + 左侧分隔线` 方案，移除了会干扰横向宽度计算的外溢装饰。
- 修复树形预览横向滚到最右侧时，路径与右边界之间会多出约 `10px` 空隙的问题。

### 当前节点摘要

- 当树形行里的字符串预览被截断为 `…` 时，底部摘要会异步向 worker 取该节点的完整值再回填显示。
- 底部摘要继续保持单行省略样式，但不再直接复用树列表那份已经被截断过的源码文本。

## 1.0.1 - 2026-03-20

### 重点更新

- 编辑区新增 `完整 / 精简` 双模式切换，大文本输入时可以直接切到更轻量的输入模式。
- 大文件场景补充了更明确的处理中反馈：完整模式粘贴大 JSON、精简模式切回完整模式时，编辑区会显示独立 loading 遮罩。
- 搜索功能继续增强：支持回车跳到下一个匹配、`Shift + Enter` 跳到上一个匹配，并补齐大数据下的搜索状态提示。
- 预览搜索性能优化：大文件 `搜预览` 改为延后调度，缓存结构化预览的可见行和命中集合，降低输入与“下一个匹配”时的整页卡顿。

### 预览与搜索体验

- 树形预览与文本预览共用命中结果，保持节点高亮、当前匹配定位和上下跳转的一致性。
- 树形与文本预览的可见区改为按缓存复用，避免仅切换当前命中时反复重建整棵可见树。
- 修复“展开全部 / 折叠全部”在结构化预览缓存开启后可能失效的问题，批量展开和折叠现在会先刷新缓存再重建预览。
- 搜索处理中会显示“搜索更新中 / 正在更新大文件搜索结果…”，并暂时禁用上下匹配按钮，避免用户误以为结果已经更新完毕。
- 搜索编辑区时不再顺带重绘整块预览无关内容；搜索预览时也不再无意义重绘编辑区高亮和行号。

### 大文件与性能

- 大文件解析节奏继续按文本规模动态延后，减少连续输入或一次性粘贴时的重复计算。
- 编辑区附属视图刷新拆成延后执行，优先让 loading 先进屏，降低“页面卡住但没有反馈”的体感问题。
- 树形预览底部保留“已经到底啦～”尾标，便于在超长数据下确认确实滚到了末尾。

### 界面与交互

- 顶部标题栏更新为 `design by Eko.Li`。
- 编辑区和预览区的搜索、复制、折叠、选择等交互继续做了细节修正，减少误触和状态不一致。
- 搜索、折叠与节点选择相关的轻提示和状态反馈继续补齐。

## 1.0.0 - 2026-03-18

### 初始版本

- 发布基于 Chrome Manifest V3 的离线 JSON 工作台。
- 提供格式化、压缩、树形预览、文本预览、元数据视图、下载、复制、导入、排序、节点复制、虚拟滚动和错误定位等核心能力。
